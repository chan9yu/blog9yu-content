Next.js 15의 주요 변경사항과 새로운 기능들을 실전 예제와 함께 알아봅니다.

React 19 지원, 성능 개선, 그리고 개발자 경험을 향상시키는 다양한 기능들이 추가되었습니다.

# 1. React 19 지원

Next.js 15는 React 19를 공식 지원합니다.

## React Compiler (Experimental)

React 19의 새로운 컴파일러는 `useMemo`, `useCallback`, `React.memo`를 자동으로 최적화합니다.

```js
// next.config.js
module.exports = {
  experimental: {
    reactCompiler: true,
  },
};
```

이제 수동 메모이제이션 없이도 자동 최적화가 가능합니다!

```jsx
// Before (React 18)
const Component = React.memo(({ data }) => {
  const processed = useMemo(() => processData(data), [data]);
  const handleClick = useCallback(() => {}, []);

  return <div>{processed}</div>;
});

// After (React 19 with Compiler)
function Component({ data }) {
  const processed = processData(data);
  const handleClick = () => {};

  return <div>{processed}</div>;
}
```

## Server Actions 안정화

React 19의 Server Actions가 안정화되어 폼 처리가 훨씬 간편해졌습니다.

```jsx
// app/actions.ts
'use server';

export async function createPost(formData: FormData) {
  const title = formData.get('title');
  const content = formData.get('content');

  await db.posts.create({ title, content });
  revalidatePath('/posts');
}

// app/new/page.tsx
import { createPost } from './actions';

export default function NewPost() {
  return (
    <form action={createPost}>
      <input name="title" />
      <textarea name="content" />
      <button type="submit">작성</button>
    </form>
  );
}
```

# 2. 비동기 Request APIs

Next.js 15에서는 `params`, `searchParams`, `cookies`, `headers` 등이 비동기로 변경되었습니다.

## Before (Next.js 14)

```tsx
// app/blog/[slug]/page.tsx
export default function Page({ params }: { params: { slug: string } }) {
  return <h1>{params.slug}</h1>;
}
```

## After (Next.js 15)

```tsx
// app/blog/[slug]/page.tsx
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  return <h1>{slug}</h1>;
}
```

## searchParams도 동일하게 변경

```tsx
export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ q: string }>;
}) {
  const { q } = await searchParams;
  const results = await search(q);

  return <Results data={results} />;
}
```

# 3. Turbopack (Stable)

Turbopack이 안정화되어 개발 서버 성능이 대폭 향상되었습니다.

```bash
# 개발 서버에서 Turbopack 사용
next dev --turbo
```

**성능 개선:**
- 로컬 서버 시작: **최대 76.7% 빠름**
- 코드 업데이트 반영: **최대 96.3% 빠름**
- Fast Refresh: **최대 45.8% 빠름**

```js
// next.config.js
module.exports = {
  // Turbopack 기본 활성화
  experimental: {
    turbo: {
      // Turbopack 설정
    },
  },
};
```

# 4. 향상된 캐싱 제어

## fetch 기본 동작 변경

```tsx
// Next.js 14: 기본 캐싱
const data = await fetch('https://api.example.com/data');

// Next.js 15: 기본 no-cache
const data = await fetch('https://api.example.com/data');

// 캐싱이 필요하면 명시적으로 지정
const data = await fetch('https://api.example.com/data', {
  cache: 'force-cache',
});
```

## unstable_cache API

서버 함수의 결과를 캐싱할 수 있습니다.

```tsx
import { unstable_cache } from 'next/cache';

const getCachedPosts = unstable_cache(
  async () => {
    return await db.posts.findMany();
  },
  ['posts'],
  { revalidate: 3600 } // 1시간
);

export default async function Posts() {
  const posts = await getCachedPosts();
  return <PostList posts={posts} />;
}
```

# 5. 부분 프리렌더링 (Experimental)

정적 콘텐츠와 동적 콘텐츠를 조합한 하이브리드 렌더링이 가능합니다.

```tsx
export default async function ProductPage({ params }: Props) {
  // 정적 부분
  const product = await getProduct(params.id);

  return (
    <div>
      {/* 정적: 빌드 타임에 생성 */}
      <ProductInfo product={product} />

      {/* 동적: 요청 시 생성 */}
      <Suspense fallback={<ReviewsSkeleton />}>
        <DynamicReviews productId={params.id} />
      </Suspense>
    </div>
  );
}
```

```js
// next.config.js
module.exports = {
  experimental: {
    ppr: true, // Partial Prerendering
  },
};
```

# 6. Instrumentation API 개선

애플리케이션 라이프사이클을 더 세밀하게 제어할 수 있습니다.

```ts
// instrumentation.ts (루트)
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    // Node.js 런타임에서만 실행
    await import('./monitoring');
  }

  if (process.env.NEXT_RUNTIME === 'edge') {
    // Edge 런타임에서만 실행
    await import('./edge-monitoring');
  }
}
```

3. ✅ **비동기 Request APIs** - 더 명확한 비동기 처리
4. ✅ **향상된 캐싱 제어** - 명시적 캐싱 전략
5. ✅ **부분 프리렌더링** - 정적 + 동적 하이브리드 렌더링
